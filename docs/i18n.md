# Localization Plan

This document records the localization structure for the IBM Z HEX ON Editor.

## Target Languages

Initial release targets:

- English
- Traditional Chinese
- Simplified Chinese
- Japanese
- Korean
- German

## Current Foundation

The first localization pass keeps English as the baseline while moving user-facing strings behind stable lookup points:

- `package.nls.json`
  - VS Code contribution strings from `package.json`, including display name, command labels, custom editor label, and configuration descriptions.
- `src/i18n.ts`
  - Extension host prompts, warnings, status messages, edit labels, encoding picker copy, and encoding descriptions.
  - Uses `vscode.l10n.t(...)` so translated VS Code bundle files can be added later without changing extension logic.
- `webview/src/i18n.ts`
  - Webview-only labels, button hints, status labels, diagnostics summary copy, and ARIA labels.
  - Uses a small catalog wrapper for now. The catalog can be replaced with locale-specific message loading when translated bundles are added.

## Follow-Up Translation Files

When the UI wording has stabilized, add translated VS Code contribution files:

- `package.nls.zh-tw.json`
- `package.nls.zh-cn.json`
- `package.nls.ja.json`
- `package.nls.ko.json`
- `package.nls.de.json`

For extension host strings, add VS Code localization bundle files generated from `vscode.l10n.t(...)` messages. Keep source strings stable while translations are being reviewed.

For webview strings, extend `webview/src/i18n.ts` to select a locale catalog from a message passed by the extension host. The webview should continue to fall back to English for missing keys.

## Stable Identifiers

Do not translate internal diagnostic identifiers such as `DBCS_AMBIGUOUS`, `MISSING_SO`, or `INVALID_OR_UNKNOWN`. Translate the labels, prompts, explanations, and UI headings that present those identifiers to users.

## Review Notes

- Leave enough layout room for German text expansion.
- Keep CJK text readable in compact and condense modes.
- Prefer explicit glossary terms for `SO`, `SI`, `DBCS`, `SBCS`, and `HEX ON`.
- Review save/reload confirmation messages carefully because they describe data-loss and raw-byte write behavior.
