# Localization Plan

This document records the localization structure for the IBM Z HEX ON Editor.

## Target Languages

Initial release targets:

- English
- Traditional Chinese
- Simplified Chinese
- Japanese
- Korean
- German

## Current Implementation

English remains the source baseline. The extension now includes first-pass catalogs for Traditional Chinese, Simplified Chinese, Japanese, Korean, and German:

- `package.nls.json`
  - VS Code contribution strings from `package.json`, including display name, command labels, custom editor label, and configuration descriptions.
  - Localized files: `package.nls.zh-tw.json`, `package.nls.zh-cn.json`, `package.nls.ja.json`, `package.nls.ko.json`, and `package.nls.de.json`.
- `src/i18n.ts`
  - Extension host prompts, warnings, status messages, edit labels, encoding picker copy, and encoding descriptions.
  - Uses `vscode.l10n.t(...)` with `package.json` `l10n` pointing to `./l10n`.
  - Localized bundle files: `l10n/bundle.l10n.zh-tw.json`, `l10n/bundle.l10n.zh-cn.json`, `l10n/bundle.l10n.ja.json`, `l10n/bundle.l10n.ko.json`, and `l10n/bundle.l10n.de.json`.
- `webview/src/i18n.ts`
  - Webview-only labels, button hints, status labels, diagnostics summary copy, and ARIA labels.
  - The extension host passes `vscode.env.language` to the webview through `EditorViewSettings.locale`.
  - Missing webview keys fall back to English.

## Stable Identifiers

Do not translate internal diagnostic identifiers such as `DBCS_AMBIGUOUS`, `MISSING_SO`, or `INVALID_OR_UNKNOWN`. User-facing diagnostic labels can be translated while preserving technical terms such as `SO`, `SI`, `DBCS`, `SBCS`, and `HEX ON`.

## Follow-Up Review

These translations are first-pass engineering translations and should be reviewed by native speakers or product localization owners before external publication.

- Leave enough layout room for German text expansion.
- Keep CJK text readable in compact and condense modes.
- Prefer explicit glossary terms for `SO`, `SI`, `DBCS`, `SBCS`, and `HEX ON`.
- Review save/reload confirmation messages carefully because they describe data-loss and raw-byte write behavior.
- Keep source English strings stable while translation review is in progress, because `vscode.l10n.t(...)` uses the source string as the lookup key.
